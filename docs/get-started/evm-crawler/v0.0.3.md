# @easylayer/evm-crawler

A self-hosted framework for building custom blockchain state services on Ethereum and EVM-compatible chains.  
Define what data you care about, point it at an RPC endpoint, and get a live + historical event stream with automatic reorg handling.

Built on **Event Sourcing + CQRS**. Ships **CJS and ESM** — runs in Node.js servers and Electron desktop apps.

---

## Installation

```bash
npm install @easylayer/evm-crawler
# or
yarn add @easylayer/evm-crawler
```

**Requirements:** Node.js ≥ 20 · TypeScript (recommended) · EVM RPC endpoint (your node or a provider like Alchemy, Infura, QuickNode)

---

## How It Works

1. You define a **Model** — a class or declarative object that says what state to track.
2. `bootstrap()` starts the crawler. It connects to your EVM node, syncs history from the configured start height, then follows new blocks in real time.
3. As blocks arrive, your model's processing function is called. You emit events via `applyEvent()`.
4. Events are persisted in the EventStore and streamed to clients over the transport of your choice.
5. Clients query the current model state or subscribe to live events via [`@easylayer/transport-sdk`](https://www.npmjs.com/package/@easylayer/transport-sdk).

## Quick Start

### 1. Create a model

Track ERC-20 Transfer events for a specific contract:

```ts
// model.ts
import type { DeclarativeModel } from '@easylayer/evm-crawler';

const TRANSFER_TOPIC = '0xddf252ad1be2c89b69c2b068fc378daa952ba7f163c4a11628f55a4df523b3ef';
const CONTRACT = '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48'; // USDC on Ethereum

export const UsdcTransferModel: DeclarativeModel<any> = {
  modelId: 'usdc-transfers',

  state: {
    transfers: [] as Array<{ from: string; to: string; value: string; blockNumber: number }>,
    count: 0,
  },

  sources: {
    async log(ctx) {
      if (ctx.log.address.toLowerCase() !== CONTRACT.toLowerCase()) return;
      if (ctx.log.topics[0] !== TRANSFER_TOPIC) return;
      return {
        from: '0x' + ctx.log.topics[1].slice(26),
        to: '0x' + ctx.log.topics[2].slice(26),
        value: BigInt(ctx.log.data).toString(),
      };
    },

    async block(ctx) {
      const logs = ctx.locals.log;
      if (!logs?.length) return;
      ctx.applyEvent('Transfer', ctx.block.number, { logs });
    },
  },

  reducers: {
    Transfer(state, event) {
      for (const log of event.payload.logs ?? []) {
        state.transfers.push({ ...log, blockNumber: event.blockHeight });
        state.count++;
      }
    },
  },
};
```

### 2. Bootstrap

```ts
// main.ts
import { bootstrap } from '@easylayer/evm-crawler';
import { UsdcTransferModel } from './model';

bootstrap({ Models: [UsdcTransferModel] });
```

### 3. Configure via `.env`

```bash
# Network
NETWORK_CHAIN_ID=1                          # 1=Ethereum, 56=BSC, 137=Polygon, etc.
PROVIDER_NETWORK_RPC_URLS=https://mainnet.infura.io/v3/YOUR_KEY

# What to index
START_BLOCK_HEIGHT=18000000                 # omit to start live-only

# EventStore (default: SQLite)
EVENTSTORE_DB_TYPE=sqlite

# Transport
TRANSPORT_HTTP_HOST=0.0.0.0
TRANSPORT_HTTP_PORT=3000
```

### 4. Query the state

```bash
curl -X POST http://localhost:3000/query \
  -H "Content-Type: application/json" \
  -d '{"name":"GetModelsQuery","dto":{"modelIds":["usdc-transfers"]}}'
```

## Models

### Declarative Model

Define state, sources (per log/transaction/block), and reducers separately.

```ts
import type { DeclarativeModel } from '@easylayer/evm-crawler';

const EthBalanceModel: DeclarativeModel<any> = {
  modelId: 'eth-balances',

  state: {
    balances: new Map<string, bigint>(),
  },

  sources: {
    async transaction(ctx) {
      // ctx.transaction, ctx.receipt available
      const { from, to, value } = ctx.transaction;
      if (!to) return; // contract creation
      return { from, to, value };
    },

    async block(ctx) {
      const txs = ctx.locals.transaction;
      if (txs?.length) {
        ctx.applyEvent('Transfer', ctx.block.number, { txs });
      }
    },
  },

  reducers: {
    Transfer(state, event) {
      for (const { from, to, value } of event.payload.txs ?? []) {
        const val = BigInt(value ?? 0);
        state.balances.set(from, (state.balances.get(from) ?? 0n) - val);
        state.balances.set(to, (state.balances.get(to) ?? 0n) + val);
      }
    },
  },
};
```

### Class-Based Model

```ts
import { Model } from '@easylayer/evm-crawler';

export class ContractEventTracker extends Model {
  static override modelId = 'contract-events';

  public events: Array<{ topic: string; data: string; block: number }> = [];
  private targetContract = '0xYourContract';

  async processBlock(ctx: any) {
    const matching = (ctx.block.logs ?? []).filter(
      (log: any) => log.address.toLowerCase() === this.targetContract.toLowerCase()
    );

    if (matching.length) {
      this.applyEvent('LogReceived', ctx.block.number, { logs: matching });
    }
  }

  protected onLogReceived(e: any) {
    for (const log of e.payload.logs) {
      this.events.push({ topic: log.topics[0], data: log.data, block: e.blockHeight });
    }
  }
}
```

## Supported Networks

The crawler is generic — it doesn't hardcode network presets. You configure the chain via `NETWORK_CHAIN_ID`.

| Network | Chain ID |
|---|---|
| Ethereum mainnet | 1 |
| BSC (BNB Smart Chain) | 56 |
| Polygon | 137 |
| Arbitrum One | 42161 |
| Optimism | 10 |
| Base | 8453 |
| Any EVM-compatible chain | set `NETWORK_CHAIN_ID` |

## EVM-Specific Features

### Receipts Strategy

Configure how the crawler fetches transaction receipts:

```bash
RECEIPTS_STRATEGY=auto            # tries eth_getBlockReceipts, falls back to per-tx
# RECEIPTS_STRATEGY=block-receipts      # strict batch mode
# RECEIPTS_STRATEGY=transaction-receipts # strict per-transaction mode
```

`auto` is recommended for most providers. Use strict modes when you need to guarantee a specific method.

### Trace Support

Enable trace data (requires a provider that supports `debug_traceBlock` or `trace_block`):

```bash
TRACES_ENABLED=true
```

The crawler checks provider capability at startup and fails fast if traces aren't available — no silent missing data.

### Mempool Monitoring

Optional. Enable to track pending transactions before block inclusion:

```bash
MEMPOOL_ENABLED=true
PROVIDER_MEMPOOL_RPC_URLS=https://your-mempool-endpoint
```

## Key Configuration Reference

### Network

| Variable | Description | Default |
|---|---|---|
| `NETWORK_CHAIN_ID` | EVM chain ID (1=Ethereum, 56=BSC, etc.) | required |
| `PROVIDER_NETWORK_RPC_URLS` | Comma-separated RPC URLs | required |
| `START_BLOCK_HEIGHT` | Block height to start from (omit for live-only) | — |
| `MAX_BLOCK_HEIGHT` | Stop processing at this height | ∞ |
| `NETWORK_PROVIDER_TYPE` | `rpc` or `ws` | `rpc` |

### Receipt & Trace

| Variable | Description | Default |
|---|---|---|
| `RECEIPTS_STRATEGY` | `auto` \| `block-receipts` \| `transaction-receipts` | `auto` |
| `TRACES_ENABLED` | Enable trace data fetching | `false` |

| Variable | Description | Default |
|---|---|---|
| `EVENTSTORE_DB_TYPE` | `sqlite` \| `postgres` | `sqlite` |
| `EVENTSTORE_DB_NAME` | SQLite: folder path; Postgres: database name | `eventstore` |
| `EVENTSTORE_DB_HOST` | Postgres host | — |
| `EVENTSTORE_DB_PORT` | Postgres port | — |
| `EVENTSTORE_DB_USERNAME` | Postgres username | — |

## Client Integration

Use [`@easylayer/transport-sdk`](https://www.npmjs.com/package/@easylayer/transport-sdk) to connect from any environment.

```ts
import { Client } from '@easylayer/transport-sdk';

// HTTP — request/response queries
const client = new Client({
  transport: {
    type: 'http',
    inbound: { webhookUrl: 'http://0.0.0.0:4000/events', pongPassword: 'pw' },
    query: { baseUrl: 'http://localhost:3000' },
  },
});

client.subscribe('Transfer', (evt) => console.log('Transfer:', evt.payload));
const result = await client.query('GetModelsQuery', { modelIds: ['usdc-transfers'] });

// WebSocket — real-time streams
const wsClient = new Client({
  transport: { type: 'ws', options: { url: 'ws://localhost:3001', pongPassword: 'pw' } },
});
```

See the [Transport SDK docs](/docs/get-started/transport-sdk) for all transport options.

## Query API

Same query API as bitcoin-crawler:

- **`GetModelsQuery`** — current model state
- **`FetchEventsQuery`** — paginated event history with filtering
- **`GetNetworkStatsQuery`** — chain sync status and stats
- **`GetNetworkLastBlockQuery`** — latest indexed block

```bash
# Get current state
curl -X POST http://localhost:3000/query \
  -H "Content-Type: application/json" \
  -d '{"name":"GetModelsQuery","dto":{"modelIds":["usdc-transfers"]}}'

# Get events filtered by block height
curl -X POST http://localhost:3000/query \
  -H "Content-Type: application/json" \
  -d '{"name":"FetchEventsQuery","dto":{"modelIds":["usdc-transfers"],"filter":{"blockHeight":18500000}}}'
```

## System Events

The crawler emits built-in events regardless of your models:

- `EvmNetworkInitializedEvent` — crawler started
- `EvmNetworkBlocksAddedEvent` — new confirmed blocks processed
- `EvmNetworkReorganizedEvent` — reorg detected and rolled back
- `EvmMempoolRefreshedEvent` — mempool snapshot (when enabled)
